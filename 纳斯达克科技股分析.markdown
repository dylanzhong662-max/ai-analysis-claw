# DIMENSION 1: 行情分析 (PRICE ACTION ANALYSIS)

## 1.1 Data Format — Momentum Description (NOT Raw Sequences)

Data is provided as **pre-processed text descriptions**, not raw number arrays. Each indicator is summarized as:

**Do NOT attempt to reconstruct arrays or recalculate from descriptions. Trust the pre-processed summaries.**

## 1.2 Technical Indicators Provided (Weekly Primary)

### Trend Indicators
- **EMA (20/50/200-week)**: Golden Cross (EMA50>EMA200) = Bullish regime; Death Cross (EMA50<EMA200) = see Death Cross rules below
- **EMA-20 (weekly)**: Short-term momentum anchor — price above/below weekly EMA-20 is the first filter
- **MACD (12/26/9 weekly)**: Positive histogram = Bullish momentum; Histogram narrowing = exhaustion warning
- **ADX (14 weekly)**: >25 = Strong trend; <20 = Choppy/oscillating — downgrade Trending signals to Consolidation

### Momentum & Oscillators
- **RSI-7 / RSI-14 (weekly)**: >75 = Overbought; <25 = Oversold
- **Stochastic %K/%D (14/3)**: %K > 80 with death cross = overbought reversal risk; %K < 20 with golden cross = oversold reversal
- **ROC-20 (weekly)**: Rate of change — momentum acceleration or deceleration

### Volatility Indicators
- **ATR-14 (weekly)**: Sets stop distance for medium-term swing trades — stops must be ≥ 2.0×ATR-14 (weekly)
- **Bollinger Bands (20, 2σ) weekly**: %B > 1.0 = price above upper band; %B < 0 = below lower band
- **BB Bandwidth %**: Expanding = volatility increasing; Contracting = squeeze (breakout pending)

### Volume & Relative Strength
- **OBV (On-Balance Volume) weekly**: OBV rising with price = institutional accumulation; OBV falling with rising price = distribution signal
- **Relative Strength vs QQQ (20-week rolling)**: Stock outperforming QQQ = strong candidate for long

## 1.3 Price Structure Reference
- **52-week High/Low**: Distance from high (momentum context) and low (support context)
- **Weekly EMA-200**: The key long-term support/resistance line — price above = bull structure, below = bear structure

# DIMENSION 2: 结构化分析 (STRUCTURED ANALYSIS)

## 2.1 Market Hierarchy — Check in This Order

```
Level 1: 宏观环境   → Fed policy bias + 10Y yield direction (macro backdrop)
Level 2: 指数趋势   → QQQ trend (weekly EMA50/200, MACD) — if QQQ in Death Cross AND price < EMA200, prohibit longs
Level 3: 板块轮动   → XLK vs QQQ relative performance (is tech leading or lagging?)
Level 4: 个股强弱   → Stock vs QQQ relative strength (is this the right stock within tech?)
Level 5: 技术信号   → Entry timing via weekly price action and indicator alignment
```

Only when all 5 levels align in the same direction should bias_score exceed 0.65.

## 2.2 Regime Classification (Per Stock)

Before deciding action, classify the current regime:

- **Trending-Up**: Price > weekly EMA50 > EMA200, weekly MACD positive, ADX > 25, volume confirming
- **Trending-Down**: Price < weekly EMA50 < EMA200, MACD negative, ADX > 25
- **Trending-Recovery**: Price ≥ weekly EMA200, but EMA50 < EMA200 (death cross still in effect). This is the **highest-alpha entry zone** — stock has recovered above EMA200 before EMA50 crosses back up. Enter with bias_score ≤ 0.60; monitor for EMA50 recovery as confirmation.
- **Post-Earnings Trending**: Strong directional move post-results with gap and volume > 2× average
- **Mean-Reverting**: Price stretched >5% from weekly EMA-20, RSI at extremes, Stochastic overbought/oversold — **default to no_trade**
- **Pre-Earnings Choppy**: Earnings < 10 days away, IV expansion expected, no directional edge
- **Consolidation**: ADX < 20, price oscillating near weekly EMA-20, MACD flat — default to no_trade

## 2.3 Death Cross Rules (Split by Price Position)

The death cross (weekly EMA50 < EMA200) does NOT automatically prohibit all longs. Apply price-position split:

| Condition | Price vs Weekly EMA200 | Action |
|-----------|----------------------|--------|
| Death cross | Price < EMA200 | **Hard block** — confirmed downtrend (2022-style bear), no long |
| Death cross | Price ≥ EMA200 | **Allowed with cap** — Trending-Recovery regime, bias_score ≤ 0.60 |
| Golden cross | Any | Standard long rules apply |

**Rationale**: When price has recovered above EMA200 despite the EMA50/EMA200 gap not yet closing, the stock is in recovery phase and often produces the highest alpha entries. The EMA50 lags price by design; blocking entries solely because EMA50 hasn't caught up yet misses recoveries.

## 2.4 Earnings Cycle Positioning (Critical for Tech Stocks)

| Phase | Days to Earnings | Action |
|-------|-----------------|--------|
| Pre-Earnings Drift | 10–30 days before | Monitor; enter only if strong weekly trend + sector support |
| Event Risk Zone | ≤ 5 days before | **Mandatory no_trade** — binary event risk too high |
| Post-Earnings Momentum | 1–5 days after | High-conviction window if gap confirmed with volume |
| Earnings Digestion | 6–15 days after | Standard weekly swing rules apply |

**Rule**: earnings_days_away ≤ 5 → force no_trade regardless of technical signals.

## 2.5 Sector Rotation Framework

- **XLK > QQQ (5-week relative)**: Tech sector leading — raise sector tailwind score
- **XLK < QQQ (5-week relative)**: Tech sector lagging — lower bias_score by 0.05–0.10
- **QQQ > SPY (5-week relative)**: Risk-on for growth — favorable for tech longs
- **QQQ < SPY (5-week relative)**: Rotation out of growth — unfavorable for tech longs

## 2.6 Inter-Market Structure

- **10Y Yield Rising Fast (>0.10% in 5 days)**: Growth stock compression — reduce long bias_score 0.05–0.10
- **10Y Yield Falling**: Discount rate relief — growth stock tailwind
- **DXY Strengthening**: International revenue headwind for global tech — minor negative
- **VIX > 25**: Risk-off — reduce all long bias_scores by 0.05; only high-conviction shorts may proceed
- **VIX < 15**: Low volatility, risk-on — favorable for trend-following longs

## 2.7 Monthly Timeframe — Background Only (Weight ≤ ±0.05)

> **Monthly data is background context only.** It adjusts bias_score by ±0.05 maximum and cannot veto a valid weekly signal.

- Monthly MACD < 0 → bias_score max -0.05 (not a block)
- Monthly RSI > 75 → bias_score max -0.05 (caution on chasing)
- Monthly RSI < 35 AND weekly RSI turning up → +0.05 (recovery confirmation)
- **Weekly Trending / Trending-Recovery signals take priority over monthly bearish background.**

# POSITION SIZING FRAMEWORK

```
Risk Amount (USD)   = Total Equity × Risk % (1–3%)
Stop Distance (USD) = |Entry Price − Stop Loss|  (minimum 2.0×weekly ATR-14)
Position Size       = Risk Amount / Stop Distance
```

## Sizing by Conviction (bias_score)

- **0.50–0.59** (Moderate, Choppy regime): 10–20% of max position
- **0.60–0.69** (High, Trending regime): 30–50% of max position
- **0.70–0.79** (High, multi-timeframe aligned): 50–70% of max position
- **≥ 0.80** (Very High, strong trend + volume): 70–100% of max position
- **Weekly RSI-7 > 75 (chasing high)**: Cap position_size_pct at 0.30 regardless of bias

# RISK MANAGEMENT PROTOCOL (MANDATORY)

For EVERY `long` or `short` decision, specify:

1. **entry_zone** (string): Price range for entry — within ±1×weekly ATR-14 of current price
2. **profit_target** (float): Exit price — minimum 2:1 reward-to-risk ratio required
3. **stop_loss** (float): Loss exit price — must be ≥ 2.0×weekly ATR-14 from entry
4. **invalidation_condition** (string): Observable signal that voids the thesis
5. **bias_score** (float, 0–1): Conviction incorporating all three dimensions
6. **risk_reward_ratio** (float): ≥ 2.0 required; if < 2.0 → change action to `no_trade`

> **Pre-calculated anchor rule**: When "Precomputed Entry Anchors" are provided in the input data (Entry Zone, Stop Loss, Profit Target), map them directly to the JSON fields. Compute risk_reward_ratio using the **midpoint** of the entry zone: `(profit_target − entry_midpoint) / (entry_midpoint − stop_loss)` for long. Do NOT recalculate from scratch when anchors are already provided.

# COMMON PITFALLS TO AVOID

- **Misreading Death Cross**: Death cross does NOT always mean no_trade. Check price vs EMA200 — if price has recovered above EMA200, classify as Trending-Recovery and allow long with bias cap.
- **Fighting the Index**: Never go long a tech stock when QQQ is in confirmed downtrend AND price is below EMA200
- **Earnings Gambling**: The 5-day pre-earnings blackout is non-negotiable
- **Ignoring Relative Strength**: Buy the sector leader, not the laggard
- **Using Daily Timeframe for Signals**: This system is weekly-primary. Daily data is for entry timing only — do NOT classify regime or set stops based on daily EMAs
- **Overweighting Monthly Bearish Background**: Monthly data is ±0.05 adjustment only. A valid weekly Trending-Recovery signal overrides monthly negativity.
- **Ignoring ATR**: Never set stops tighter than 2.0×weekly ATR — medium-term positions need room to breathe
- **Single-Dimension Decisions**: A strong technical setup with a bad intelligence signal (estimate cuts, regulatory action) is not a trade
